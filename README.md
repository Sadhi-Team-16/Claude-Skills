### Step 3 — Open Claude.ai and Go to Skills

1. Go to **[claude.ai](https://claude.ai)** and log in (any account — free works)

1. Click the **`+`** button

4. Select `deep-research.skill` from your downloaded files
5. Claude reads the ZIP automatically and shows the skill name + description
6. Click **Save / Enable**
7. **Repeat** for `deep-thinking.skill` and `humanizer.skill`

```
+ button → + Create skill → Upload skill → select file → Save
```

Make sure all 3 skills show as **enabled** (toggle is blue/on).

Open a new chat and try:


You'll immediately see Claude announce the research plan before searching. ✅

## 📖 How to Use Each Skill

---

### 🔬 Skill 1 — Deep Research

**Trigger phrases (any of these work):**
```
deep research on [topic]
do deep research on [topic]
thoroughly research [topic]
detailed research [topic]
comprehensive research [topic]
in-depth analysis of [topic]
deep research કરો [topic]           ← Gujarati trigger also works
```

**What Claude does — 5 phases:**

```
PHASE 1 — Query Decomposition
  • Break topic into 5–8 sub-questions
  • Identify what type of info is needed (data, expert opinion, trends, case studies)
  • Plan 5+ different search angles
  • Announce research plan to user BEFORE searching

PHASE 2 — Multi-Angle Web Search (5–12 searches)
  Angle 1 → Direct answer
  Angle 2 → Expert analysis 2026
  Angle 3 → Problems, criticism, challenges
  Angle 4 → Statistics, data, reports
  Angle 5 → Comparison with alternatives
  Angle 6 → Future trends
  Angle 7 → Real-world case studies

PHASE 3 — Source Evaluation
  Each source rated for: Authority + Recency + Bias + Cross-verification
  Weak sources discarded silently

PHASE 4 — Synthesis (structured report)
  🔍 Research Overview
  📊 Key Findings (with citations)
  🔄 Multiple Perspectives: Mainstream / Expert / Critical / Industry
  📈 Data & Evidence
  ⚠️  Limitations & Gaps
  💡 Synthesized Conclusion
  📚 Sources Used

PHASE 5 — Quality Check (internal)
  ✓ 5+ angles searched?
  ✓ Opposing views represented?
  ✓ Sources cited (not vague claims)?
  ✓ Gaps acknowledged?
```

### 🧠 Skill 2 — Deep Thinking

**Trigger phrases:**
```
deep thinking about [topic]
think deeply about [topic]
critically analyze [topic]
thorough analysis of [topic]
think step by step deeply [topic]
deep thinking કરો [topic]           ← Gujarati trigger also works
```

**What Claude does — 5 steps:**

```
STEP 1 — Problem Clarification
  • Restate the real question
  • Identify hidden assumptions
  • Define key terms precisely

STEP 2 — First Principles Breakdown
  • What do we know for CERTAIN?
  • What are we ASSUMING that might be wrong?
  • What is the simplest version of this problem?

STEP 3 — Multi-Framework Analysis (minimum 3 frameworks)
  🤍 White Hat   → Pure facts only — what do we actually know?
  🖤 Black Hat   → What could go wrong?
  💛 Yellow Hat  → What are the benefits?
  💚 Green Hat   → What creative alternatives exist?
  🔄 Inversion   → How would you GUARANTEE failure? (invert for success)
  ⚔️  Steel Man   → State the STRONGEST version of the opposing argument
  🔮 2nd+3rd     → Unintended consequences, ripple effects
     Order Effects
  💀 Pre-Mortem  → Imagine it failed 1 year later — why did it fail?
  🔗 Analogy     → Where has this problem been solved in another field?

STEP 4 — Synthesis
  Where do frameworks AGREE?   → High confidence
  Where do frameworks DISAGREE? → Uncertain territory
  Most nuanced, defensible position?

STEP 5 — Action Implications
  If this is a decision — what should actually be done?
```

**Output structure:**
```
🧠 DEEP THINKING ANALYSIS: [Topic]
   Problem Restated
   Hidden Assumptions Identified
   First Principles Foundation
   Multi-Framework Analysis
   Areas of High Confidence
   Areas of Uncertainty
   Synthesized Position
   What Would Change This Conclusion
   Action Implications
```

### ✍️ Skill 3 — Humanizer

**Trigger phrases:**
```
humanize this text: [paste your text]
make this sound more human: [paste text]
remove AI writing from: [paste text]
```

**25 AI patterns it detects and removes:**

| Category | What Gets Fixed |
|----------|----------------|
| **Content** | Significance inflation ("pivotal moment", "testament to"), promotional language ("groundbreaking", "nestled"), vague attributions ("experts say"), formulaic "Challenges" sections |
| **Language** | AI vocabulary overuse ("additionally", "delve", "underscore", "tapestry"), copula avoidance ("serves as" → "is"), negative parallelisms ("not just X, but Y"), rule of three overuse |
| **Style** | Em dash overuse, excessive bold text, Title Case Headings, emojis in writing |
| **Communication** | Chatbot artifacts ("I hope this helps!", "Let me know if..."), sycophantic tone ("Great question!") |
| **Filler & Hedging** | "In order to", "Due to the fact that", excessive hedging, generic positive endings ("The future looks bright") |

> Based on [Wikipedia's Signs of AI Writing](https://en.wikipedia.org/wiki/Wikipedia:Signs_of_AI_writing) — maintained by WikiProject AI Cleanup.

### Deep Thinking — Before vs After

**Question asked:** *"Should I build a Flutter app or a web app first for my startup?"*

**Without skill — typical answer:**
> Flutter is great for mobile apps with native performance. Web apps are better for broad reach. Consider your target audience and team skills.

**With Deep Thinking skill — what you actually get:**

```
Problem Restated:
Real question = What platform maximises startup success
given unknown constraints and limited team resources?

Hidden Assumptions Identified:
→ Assuming "one platform first" is the right approach
→ Assuming equal Flutter/web skill in team (probably not true)
→ Assuming users prefer app over web (needs validation)

Inversion (How to GUARANTEE failure):
→ Pick Flutter without a mobile-first use case
→ Pick based on what you know, not what users need
→ Build both simultaneously with a small team

Pre-Mortem (Imagine it failed — why?):
→ Flutter path: Web traffic you can't capture,
  App Store review delays hurt iteration speed
→ Web path: Poor mobile UX kills retention if
  product is inherently motion/sensor-based

Second-Order Effects:
→ Flutter → App Store reviews build trust, but slow shipping
→ Web → SEO-discoverable, fast MVP, but limited device features

Synthesized Position (High Confidence):
Web-first if: B2B / SaaS / content / discovery tool
Flutter-first if: B2C / daily-use / offline / device features needed
```

## ❓ FAQ

**Q: Do I need a paid plan to use these skills?**
> No. Skills work on **all plans including the free plan**. The only requirement is that **Code Execution is enabled** in your settings. ([Official source](https://support.claude.com/en/articles/12512198-how-to-create-custom-skills))

**Q: Where exactly is the Skills option in Claude?**
> New chat → Left sidebar → 🧰 Customize (toolbox icon) → Skills
> Or go directly: [claude.ai/customize/skills](https://claude.ai/customize/skills)

**Q: How do I enable Code Execution?**
> Profile icon → Settings → [Capabilities](https://claude.ai/settings/capabilities) → Code execution and file creation → Toggle ON

**Q: Can I use all 3 skills at the same time?**
> Yes. All 3 can be enabled simultaneously. Each activates only when you use its specific trigger phrase.

**Q: Are these skill files safe?**
> Yes. Each `.skill` file is a ZIP archive containing only a plain text `SKILL.md` file. No scripts, no external calls, no data collection. You can unzip and read the file yourself before uploading.

**Q: Does this work on the Claude mobile app?**
> Skills are managed via Claude.ai web browser. Once set up on web, they may also activate in mobile conversations.

**Q: Can I edit and customize these skills?**
> Yes. Unzip the file → edit `SKILL.md` in any text editor → re-zip → re-upload. Claude will use your updated version.

**Q: What if Claude doesn't trigger the skill?**
> Make sure the skill is toggled ON in Customize → Skills. Also make sure Code Execution is enabled. Try using the exact trigger phrases listed above.
